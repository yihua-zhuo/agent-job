### Key design decisions

- **All 10 methods are `async def`** — callers in `src/api/sales.py` (Flask) will need `async` route handlers or a sync-to-async bridge (e.g. `asyncio.run()` at the route level, or FastAPI). The method signatures and return types (`ApiResponse`) are unchanged.
- **`async with get_db_session() as session`** — used in every method; the context manager handles commit on success and rollback on exception.
- **Creates** use `session.add()` + `session.flush()` to get the DB-assigned `id` (equivalent to `RETURNING`).
- **Updates** use `update(Model).where(id=..., tenant_id=...)` so tenant isolation is enforced at the SQL level.
- **Stats / funnel / forecast** are replaced with `func.count` / `func.sum` + `GROUP BY` — much more efficient than loading all rows into Python.
- **Chinese docstrings and error messages** are preserved throughout.
- `delete` and `text` are imported as requested (unused but present for completeness).
