## 1. 目标与背景

### 1.1 为什么做

Issue #449 introduced the workflow system; issue #461 built `WorkflowService` with CRUD and `execute` methods. However, there is currently no persisted record of workflow executions — each `execute` call writes nothing to the DB, making audit, replay, and error-diagnosis impossible. Without an execution history table the automation engine cannot support retry, branching logic, or observability features planned in later subtasks.

### 1.2 做完后

- **用户视角**：无用户可见 changes — pure backend infrastructure.
- **开发者视角**：`WorkflowExecution` ORM model is importable from `src.db.models.workflow_execution`; `WorkflowService.execute` can persist `input_data`, `output_data`, `status`, and `error_message` to the DB. Any downstream service can query past executions via `WorkflowExecutionService`.

### 1.3 不做什么（剔除）

- [ ] No API router for execution history (deferred to a later subtask of #449)
- [ ] No retry / replay logic — writes to the table only, consumption by downstream services is out of scope
- [ ] No CSV / export endpoint

### 1.4 关键 KPI

- [KPI 1：`alembic upgrade head` → exit 0, `alembic downgrade -1` → exit 0 (migration is reversible)]
- [KPI 2：`ruff check src/db/models/workflow_execution.py` → 0 errors]

## 4. 设计决策与已知坑

### 4.1 关键选型

- **JSONB vs JSON**：`input_data` and `output_data` use `JSONB` (not `JSON`) — enables GIN index support and efficient querying of nested fields, which is required for future branching logic.
- **Status enum stored as String**：`status` is stored as `String` (not `Enum`) to avoid alembic autogenerate creating a separate enum type object, which complicates migration portability across environments. Consumers can validate against a Python `enum` at the service layer.
- **separate `started_at` + `completed_at`** over a single `duration` field — allows computing `duration` at read time and distinguishes never-started (NULL) from zero-duration executions.

### 4.2 版本约束

无新依赖引入。

### 4.3 兼容性约束

- 多租户：所有查询必须 `WHERE tenant_id = :tenant_id`
- ORM model 类名不可与 SQLAlchemy `Base.metadata` 冲突 — 禁止使用 `metadata` 作为列名列，使用 `event_metadata` 或 `payload` 等替代（本模型无此字段，纯合规说明）
- Service 层调用 `.to_dict()`，模型本身不调用
- Migration 中 `status` 列为 `String`（非 `Enum`），避免跨环境 enum 类型迁移问题

### 4.4 已知坑

1. **Alembic autogenerate emits `sa.JSON()` instead of `sa.JSONB()`** → 规避：手动将 autogenerated `Column(JSON)` 改为 `Column(JSONB)` for `input_data` 和 `output_data` 列
2. **Alembic autogenerate drops `timezone=True` on `DateTime`** → 规避：确认 `started_at` 和 `completed_at` 为 `TIMESTAMPTZ`（`DateTime(timezone=True)`），手动补上
3. **PYTHONPATH=src**：import 路径写 `from db.models.workflow_execution` 而非 `from src.db.models.workflow_execution`

## 5. 实现步骤（按顺序）

### Step 1: Create `src/db/models/workflow_execution.py`

定义 `WorkflowExecution` ORM model，包含所有列：id, tenant_id, workflow_id, triggered_by, input_data (JSONB), output_data (JSONB), status (String), started_at (DateTime), completed_at (DateTime, nullable), error_message (nullable).

代码结构参考现有 ORM 模型（如 `ticket.py` 或 `pipeline.py`），继承 `Base` from `db.base`，`__tablename__ = "workflow_execution"`。

```python
from datetime import datetime
from typing import Optional

from sqlalchemy import DateTime, Index, Integer, String, Text
from sqlalchemy.dialects.postgresql import JSONB
from sqlalchemy.orm import Mapped, mapped_column

from db.base import Base


class WorkflowExecution(Base):
    __tablename__ = "workflow_execution"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    tenant_id: Mapped[int] = mapped_column(Integer, nullable=False, index=True)
    workflow_id: Mapped[int] = mapped_column(Integer, nullable=False, index=True)
    triggered_by: Mapped[str] = mapped_column(String(255), nullable=False)
    input_data: Mapped[dict] = mapped_column(JSONB, nullable=False, default=dict)
    output_data: Mapped[dict] = mapped_column(JSONB, nullable=False, default=dict)
    status: Mapped[str] = mapped_column(String(50), nullable=False)
    started_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), nullable=False, default=datetime.utcnow
    )
    completed_at: Mapped[Optional[datetime]] = mapped_column(
        DateTime(timezone=True), nullable=True
    )
    error_message: Mapped[Optional[str]] = mapped_column(Text, nullable=True)

    __table_args__ = (
        Index("ix_workflow_execution_tenant_workflow", "tenant_id", "workflow_id"),
    )
```

from sqlalchemy import inspect

class TestWorkflowExecutionModel:

    def test_column_names(self):

            "id", "tenant_id", "workflow_id", "triggered_by",
            "input_data", "output_data", "status",
